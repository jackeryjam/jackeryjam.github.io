搭建个人博客

简介

　　本文的主要内容是如何使用hexo和github搭建个人博客，不止是coding，像github、gitee等这些代码托管的网站，都提供了page的服务，就是可以把远程仓库作为一个http服务器，这当然比我们自己购买一台服务器来搭建http服务器方便多了，还有对应的域名，省去了诸多麻烦，不过限制是一般只能提供静态页面的服务，coding比较高端一点，还提供一个256M的mysql以及支持php，这些免费提供的服务在性能及功能上自然是比自己购买服务器差多了，不过不打紧，搭建自己博客也够了(有没有人看哎不一定呢，等人多了在去考虑自己搞一个服务器)

优势

  • 免费的服务器及域名
  • hexo可以很方便地生成博客的静态网页以及部署到git仓库
  • markdown很方便

步骤

  • 使用hexo初始化博客，认识hexo的几个命令,hexo官网

    • 预置要求: 先要安装node环境
    • 全局安装hexo脚手架: npm install hexo-cli -g
    • 创建自己的文件夹(blog)，在命令行中执行hexo init进行初始化
    • 安装依赖npm install
    • 其中生成的sourse文件夹就是保存我们文章的地方，在sourse中创建对应的文件夹写markdown就可以了
    • hexo还支持分类和打标签

  • 了解认识markdown

    • markdown的话就记住一下那些简单的用法，例如标题用#表示，列表用*表示，之后就像写word文档那样，写文章就写个大标题，之后下面就几个小标题，小标题里面可能就再加些列表什么的就行了
    • 最基本朴实的文章也就是有标题和内容构成，这样也简洁大方，如果加些链接和图片，文章就变成丰富多彩的富文本了
    • markdown并不像html能够给浏览器直接解析，需要用转化位html才能在网页上美哒哒地显示出来，执行hexo g就能把你写的md转化成对应的html存放到public文件夹
    • 因为关于markdown的资料也很多，我这里也不累述，就说一下基本用法和心得体会

  • 将生成的页面部署到服务端

    • 执行hexo s可以开本地服务器进行预览，边编写边看效果，也是不错的
    • 要上传到服务器，建议用个coding新建一个对应用户名的仓库，这样通过username.coding.me这个域名访问，当然在github上新建就可以通过username.github.io访问对应用户名仓库的根目录，记得把page功能打开，这样仓库才可以当初http服务器使用，如下图操作，github的操作也差不多
    • 开启后我们手动把代码上传到git服务器，就可以有如下的效果
    • 但是这样要每次都手动地上传，这时候就会觉得有点烦人，hexo有个自动部署的，这个时候就可以，emmmmmm
      只需要把hexo生产的项目的根目录上的_config.yml里面最后的部署加上github对应的地址就可以了，如下

      deploy:
    • type: git
      repo: https://username:password@github.com/jackeryjam/jackeryjam.github.io.git
      branch: master
      加完这些东西，执行hexo d
